\label{sec:conclusion}

This paper presents a PeerJ-style application-oriented draft for evaluating and
adapting open LLMs under a single RTX 3090 24GB constraint. The project now
contains a railway bilingual QA dataset, a reproducible evaluation workflow,
compressed baseline result tables, a QLoRA adapter result table, a normalized
BibTeX library, and a JSON literature-method matrix.

The completed result export indicates that Qwen2.5-7B-Instruct and Qwen3-8B
are the strongest accuracy-oriented baselines, while Qwen3-0.6B and Qwen3-1.7B
are the most efficient lightweight choices. Qwen3-4B is a practical middle
candidate for continued QLoRA testing. The current quantization results show
clear memory savings for several int4 runs, but not consistent latency gains,
which confirms the need for measurement on the actual deployment stack.

The main unfinished empirical item is the full railway-domain evaluation. The
current Domain QA and QLoRA domain scores are one-sample sanity checks, not
final domain results. Once the 2,770-sample test split is rerun for baselines
and adapters, the manuscript can be upgraded from a pipeline-complete draft to
a full empirical paper with stronger domain conclusions.
