\label{sec:experimental_setup}

\textbf{Hardware and software.} All experiments are designed for a single
NVIDIA RTX 3090 GPU with 24GB of VRAM. The implementation uses PyTorch,
Hugging Face Transformers, PEFT, BitsAndBytes, Datasets, and the Language Model
Evaluation Harness. All scripts are executed in the foreground to keep GPU use
observable and reproducible.

\textbf{Model pool.} The formal model pool contains nine locally cached open
models. The pool intentionally excludes models that were not fully available in
the local cache, whose access authorization was rejected during preparation, or
whose architecture was not supported by the current Transformers environment.
This keeps the formal experiment reproducible on the current hardware and
software environment. \Cref{tab:model-overview} lists the final model pool.

\textbf{Railway-domain dataset.} The domain dataset is generated from local
railway bilingual corpus files under \texttt{data/corpus/railway}. The sources
include railway technical terminology and traction power supply operation and
maintenance regulations. The generated dataset contains four categories:
Chinese-to-English terminology, English-to-Chinese terminology,
Chinese-to-English regulation translation, and English-to-Chinese regulation
translation. After conservative cleaning, the dataset contains 22,164 training
examples, 2,770 validation examples, and 2,770 test examples.

\textbf{Smoke dataset.} Before formal experiments, a stratified 1\% smoke
dataset is generated from the full domain dataset. It contains 222 training
examples, 28 validation examples, and 28 test examples. This smoke dataset is
used only to validate data loading, model loading, evaluation output paths, and
the foreground execution scripts.

\textbf{Benchmark suite.} The formal benchmark suite includes public evaluation
tasks and a local railway-domain QA task. Public tasks are used to measure
general language, reasoning, code, and Chinese capabilities. The local task is
used to measure railway-domain terminology and regulation handling. The
pipeline records raw generations and summary metrics so that both quantitative
and qualitative analysis can be performed.

\textbf{Quantization settings.} Baseline inference is evaluated under bf16,
int8, and int4 settings where supported. These settings allow the paper to
compare accuracy, peak memory, latency, and throughput under the same hardware
constraint.

\textbf{QLoRA setup.} QLoRA is applied to a small representative subset:
Qwen3-4B, Qwen2.5-7B-Instruct, and Phi-3-mini-4k-instruct. The base model is
loaded in 4-bit mode, original weights are frozen, and LoRA adapters are trained
on the railway-domain dataset. The goal is not to create a new adaptation
algorithm, but to test whether low-cost domain adaptation is useful under a
single-GPU budget.
