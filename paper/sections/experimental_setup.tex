\subsection{Hardware and Software}

All reported measurements target a single NVIDIA RTX 3090 GPU with 24GB of
VRAM. The local workflow uses PyTorch, Hugging Face Transformers, PEFT,
BitsAndBytes, Datasets, and the Language Model Evaluation Harness. Result
artifacts are stored under \path{results/single_gpu_3090/}, and configuration
files are stored under \path{configs/}.

\subsection{Tasks and Metrics}

The public benchmark suite contains MMLU, C-Eval, GSM8K, and HumanEval.
Domain-QA and Domain-RegQA are evaluated with exact match, character-level F1,
token-level F1, reference containment, and length ratio. Efficiency is measured
with peak allocated VRAM, peak reserved VRAM, latency, and generated-token
throughput. For QLoRA, the training export records runtime, training loss,
evaluation loss, evaluation accuracy, and perplexity when available.

\subsection{Precision and Adaptation Settings}

The baseline plan compares bf16, int8, and int4 inference when supported. The
primary final sweep is int4 because it is the most relevant setting for fitting
multiple 4B--9B models into a 24GB deployment envelope. The completed QLoRA
configuration uses 4-bit base loading, LoRA rank 64, LoRA alpha 16, dropout
0.05, learning rate \(2\times10^{-4}\), three epochs, per-device training
batch size 1, gradient accumulation 16, and maximum sequence length 2,048.
