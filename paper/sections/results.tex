\subsection{Efficiency and Deployment Cost}

Table~\ref{tab:efficiency-results} reports peak memory, latency, and throughput under the same prompt set and generation configuration. This table is essential because deployment decisions under a 24GB budget are constrained by more than accuracy alone. In the final manuscript, the narrative should identify models that dominate in practical terms, for example by offering near-7B quality at substantially lower memory cost or by maintaining acceptable quality while significantly improving response speed.

The final analysis in this subsection should avoid two common mistakes. The first is to interpret throughput without accounting for quality. A very fast but substantially weaker model may still be a poor deployment choice. The second is to interpret quality without accounting for latency. A reasoning-oriented model may deliver better mathematical performance but remain unattractive if the latency penalty is too high for interactive use. The paper should therefore report both absolute values and ratio-style interpretations, such as task score per gigabyte and score per unit latency.

\subsection{QLoRA Adaptation Results}

Table~\ref{tab:qlora-results} compares selected models before and after QLoRA-based domain adaptation. The main purpose is to determine whether lightweight adaptation changes the deployment recommendation under the same hardware limit. If a model that is only moderately strong in the baseline becomes highly competitive after QLoRA, that result is more practically important than a small improvement on the already strongest baseline.

\subsection{Scaling and Tradeoff Analysis}

Beyond the tables, the final manuscript should include at least two figures. The first figure should plot model size against a normalized aggregate benchmark score, making visible whether capability improves smoothly or in stages. The second figure should visualize the tradeoff between quality and latency, ideally with marker size or color encoding memory footprint. This can reveal whether certain models lie on the Pareto frontier while others are clearly dominated.

If the final evidence shows that 4B-class models remain highly competitive after QLoRA, the paper should state that conclusion plainly. Such a finding would be especially important for practitioners because it suggests that careful adaptation can substitute for brute-force model scaling in many single-GPU deployment scenarios.
